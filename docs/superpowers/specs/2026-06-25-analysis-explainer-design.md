# Analysis Explainer + README — Design

**Date:** 2026-06-25
**Status:** Approved design, pre-plan
**Builds on:** `2026-06-25-run-analysis-design.md` (zone distribution, decoupling, coaching, AnalysisStrip, RunSummary). Same branch family.

## Goal

Make the run-analysis metrics self-explanatory:

1. A tappable in-app explainer for the analysis panel — explains **Drift** (aerobic decoupling) and the **Karvonen zones**, including each zone's **fuel source** (fat vs carbs), so a user can tell whether they are training to burn fat (Zone 2) or burn carbs (Zone 3+).
2. A **README** documenting the app and the same metrics.

Both read their copy from one pure-core glossary module, so the modal, the README, and the tests never drift apart.

## Non-Goals

- No changes to the analysis math (zoneDistribution / decoupling / coaching are done).
- No pace/HR/distance glossary (scope is Drift + Zones only).
- No web-only hover behaviour; tap is the cross-platform trigger.
- No metabolic fuel-split *chart* (PRD FR-5.3 stays deferred) — only a per-zone fuel label.

## Architecture

A pure-TypeScript `core/glossary.ts` holds the explainer content as data. The UI modal (`AnalysisInfoModal.tsx`) renders it, following the existing `RoutesPanel`/`StravaPanel` modal pattern (React Native core `Modal`, transparent backdrop, `Pressable` to close — NOT react-native-paper Dialog, to match the established pattern). `AnalysisStrip` and `RunSummary` gain an ⓘ affordance and an optional `onInfo` callback; `RunScreen` owns the modal's visibility state. The README is generated by hand from the same glossary concepts.

### Global constraints

- TypeScript strict; no `any` in committed code.
- `core/` modules MUST NOT import from react/react-native/expo/ui — pure and platform-agnostic.
- No new dependencies. (`MaterialCommunityIcons` from `@expo/vector-icons` is already used in the app.)
- Reuse existing `ZoneId`/`DecouplingRating` types, `ZONE_THEME` (`src/ui/theme.ts`), and the dark palette + modal pattern from `RoutesPanel.tsx`.
- Component return types use `React.JSX.Element`.
- Accessibility: ⓘ control has an accessibility label; tappable panel is a `Pressable` with a button role + label.
- New `onInfo` props are OPTIONAL (`?`) so existing call sites keep compiling.
- TDD for the core module: failing test first, minimal data, passing test, commit.

## §1 — Glossary content (pure core)

**File:** `src/core/glossary.ts`, test `src/core/__tests__/glossary.test.ts`.

**Consumes:** `ZoneId` from `./karvonen`, `DecouplingRating` from `./decoupling` (type-only imports).

**Produces:**

```ts
export interface ZoneInfo {
  id: ZoneId;
  label: string;   // e.g. "Zone 2 · Aerobic"
  range: string;   // e.g. "60–70% HRR"
  fuel: string;    // e.g. "Mostly fat"
  meaning: string; // one-line plain-language description
}

export interface DriftBand {
  rating: DecouplingRating;
  range: string;   // e.g. "< 5%"
  meaning: string;
}

export const DRIFT_INTRO: string;        // what drift is; "<5% = solid aerobic run"; the ~6-min note
export const ZONE_INFO: ZoneInfo[];      // exactly the 4 ZoneIds, in order below/zone2/zone3/above
export const DRIFT_BANDS: DriftBand[];   // exactly the 3 ratings: good, moderate, high
```

**Content (authoritative copy):**

- `DRIFT_INTRO`: "Drift (aerobic decoupling) compares the first and second half of your run. It measures how much your heart rate crept up to hold the same pace. Lower is better — under 5% is a solid, well-paced aerobic run. It needs about 6 minutes of running before it can show a number."

- `ZONE_INFO` (ranges are % of Heart-Rate Reserve, matching `zoneForHr`):
  - `below` — label "Below Zone 2", range "Under 60% HRR", fuel "Mostly fat", meaning "Very easy — warm-up or recovery pace."
  - `zone2` — label "Zone 2 · Aerobic", range "60–70% HRR", fuel "Mostly fat", meaning "Easy aerobic base. Builds endurance and fat-burning efficiency."
  - `zone3` — label "Zone 3 · Tempo", range "70–80% HRR", fuel "Fat + carbs", meaning "Comfortably hard. Shifts toward burning carbohydrate (glycogen)."
  - `above` — label "Above Zone 3", range "Over 80% HRR", fuel "Mostly carbs", meaning "Hard effort. Burns mostly carbohydrate; not for base-building."

- `DRIFT_BANDS`:
  - `good` — range "< 5%", meaning "Heart rate held steady. Strong aerobic base, well paced."
  - `moderate` — range "5–10%", meaning "Mild drift. Base still building, or you started too fast."
  - `high` — range "≥ 10%", meaning "Heart rate climbed to hold pace. Went too hard, under-trained, hot, or under-fuelled."

The carb-vs-fat fuel labels are the direct answer to "am I burning carbs?": Zone 2 = fat, Zone 3+ = carbs.

## §2 — Explainer modal (UI)

**File:** `src/ui/AnalysisInfoModal.tsx`.

**Consumes:** `DRIFT_INTRO`/`ZONE_INFO`/`DRIFT_BANDS` from `../core/glossary`; `ZONE_THEME` from `./theme`; `DecouplingRating` from `../core/decoupling` (for the rating→color map); React Native `Modal`/`View`/`ScrollView`/`Pressable`/`StyleSheet`; `Text` from react-native-paper; `MaterialCommunityIcons`.

**Produces:**

```ts
function AnalysisInfoModal(props: { visible: boolean; onClose: () => void }): React.JSX.Element
```

**Structure (follows `RoutesPanel.tsx`):** `<Modal transparent visible={visible} animationType="fade" onRequestClose={onClose}>` → full-screen `Pressable` backdrop (`rgba(0,0,0,0.65)`, closes on press) → a panel `View` (dark surface, rounded) containing a `ScrollView`:

- Title: "Understanding your run".
- **Drift** section: heading "Drift — aerobic decoupling", `DRIFT_INTRO` paragraph, then each `DRIFT_BANDS` row: a colored chip (rating→color map below) with `range` + `meaning`.
- **Zones** section: heading "Heart-rate zones & fuel", then each `ZONE_INFO` row: a `ZONE_THEME[id]` colored dot + icon, `label`, `range`, **`fuel`** (emphasised), `meaning`.
- A "Got it" close button.

Rating→color map (reuse the RunSummary palette so the app is internally consistent):
`good → "#0E7C7B"`, `moderate → "#B45309"`, `high → "#9D174D"`. Typed `Record<DecouplingRating, string>`.

Read-only and static — no engine interaction, no live data.

## §3 — Tappable affordance + wiring

**`src/ui/AnalysisStrip.tsx`:** add optional prop `onInfo?: () => void`. Wrap the strip body in a `Pressable` (accessibilityRole "button", label "What do these mean?") that calls `onInfo` when set. Add a small `MaterialCommunityIcons name="information-outline"` next to the "Drift" label. When `onInfo` is undefined the panel is inert (renders the same, no press handler).

**`src/ui/RunSummary.tsx`:** add optional prop `onInfo?: () => void`. Add the same ⓘ icon to the decoupling block header and make that block a `Pressable` calling `onInfo`.

**`src/ui/Dashboard.tsx`:** add optional pass-through prop `onInfo?: () => void`, forwarded to `<AnalysisStrip onInfo={onInfo} />`.

**`src/ui/RunScreen.tsx`:** add `const [infoOpen, setInfoOpen] = useState(false);`. Pass `onInfo={() => setInfoOpen(true)}` to `<Dashboard>` and to `<RunSummary>`. Render `<AnalysisInfoModal visible={infoOpen} onClose={() => setInfoOpen(false)} />` inside BOTH the finished/summary return branch and the main run-view return branch, driven by the shared `infoOpen` state. (A `Modal` only shows when `visible`, so having it in both branches is harmless and keeps each branch self-contained.)

All new props OPTIONAL → no existing call site breaks.
